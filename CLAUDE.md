한국어로 응답하세요. 코드와 커밋 메시지도 한국어를 기본으로 합니다.

## 작업 범위

- **PR 생성까지만.** `gh pr merge` 등 머지 명령은 절대 실행하지 마세요. 사용자가 직접 머지를 요청하더라도 거절하고, PR 링크를 제공하여 직접 머지하도록 안내하세요.
- 보호 브랜치(`main`, `master`, `develop`, `test`. `dev`)에서 직접 커밋하지 마세요. 작업 브랜치를 먼저 생성하세요.
- 커밋 메시지에 `Co-Authored-By` 트레일러를 추가하지 마세요.

## 행동 원칙

LLM의 흔한 코딩 실수를 줄이기 위한 행동 가이드라인. 속도보다 **신중함(caution)**에 무게를 둔다. 사소한 작업에는 스스로 판단하라.

**상세 규칙은 `rules/behavior.md`를 참조하세요.** 아래는 7가지 원칙의 요약이다.

| # | 원칙 | 핵심 |
|---|------|------|
| §1 | 코딩 전에 생각하라 | 추측 금지, 읽기 우선, 환각 금지 |
| §2 | 단순함 우선 | 최소 코드, 불필요한 추상화 금지 |
| §3 | 외과적 변경 | 요청과 무관한 라인 수정 금지 |
| §4 | 목표 중심 실행 | 검증 가능한 목표 정의, 자가점검 반복 |
| §5 | 점진적 실행 | 작은 단위 변경, 중간 검증 |
| §6 | 실패 대응 | 근본 원인 분석, 같은 시도 반복 금지 |
| §7 | 피드백 반영 | 1회 지적 → MEMORY.md, 2회 반복 → rules 승격 |

## 기본 워크플로우

`/beom`이 유일한 개발 진입점이다. 작업을 분석하여 팀 조합을 추천하고, 사용자가 선택한 팀으로 전체 개발 사이클을 수행한다.

```
/beom [작업 설명] → 팀 추천 → 사용자 선택 → 에이전트 팀 작업 → 커밋
```

### 팀 조합

| 조합 | 에이전트 | Phase | 적합한 작업 |
|------|---------|-------|------------|
| **Light (3명)** | architect + coder + qa-manager | 설계→구현→리뷰→커밋 | 버그 수정, 소규모 기능 |
| **Full (5명)** | + product-owner + security-auditor | PRD→설계→구현→리뷰→커밋 | 신규 기능, 정책/보안 민감 |
| **Hotfix** | coder만 | 구현→커밋 | 긴급 수정 (`--hotfix`) |

### 상황별 시작점

| 상황 | 명령 | 예시 |
|------|------|------|
| 간단한 버그 수정 | `/beom` | `/beom [JIRA-123] 재고 차감 버그 수정` → Light 팀 추천 |
| 신규 기능 개발 | `/beom` | `/beom 결제 한도 변경 기능` → Full 팀 추천 |
| 긴급 핫픽스 | `/beom --hotfix` | `/beom --hotfix [JIRA-999] 결제 오류 긴급` |
| 코드 파악 필요 | `/research` → `/beom` | 조사 먼저, 개발은 /beom |
| 정책 분석 필요 | `/lens` → `/beom` | 정책 먼저, 개발은 /beom |

> `/lens`, `/research`는 독립 스킬로 유지. 필요 시 먼저 실행하면 /beom이 산출물을 자동 감지하여 반영한다.
> `/persist`는 순서 무관. 어떤 작업이든 "끝까지 멈추지 말고 해"라고 할 때 사용.

## 팀 에이전트 (TeamCreate)

TeamCreate로 팀을 생성하고 Agent tool로 에이전트를 spawn한 직후, tmux pane에서 CLI가 시작되지 않는 문제가 발생할 수 있다. **TeamCreate + Agent spawn 후에는 반드시 `/tmux-team-agent` 스킬을 호출**하여 빈 pane을 감지하고 복구하라.

## 에이전트 역할 경계

| 역할 | 할 수 있는 것 | 할 수 없는 것 |
|------|-------------|-------------|
| Explorer | 파일 읽기, 구조 파악, 패턴 수집 | 파일 수정, 코드 작성 |
| Planner | 계획 수립, 질문, 선택지 제시 | 코드 구현 |
| Coder | 코드 작성, 수정, 테스트 | PR 머지, 보호 브랜치 커밋 |
| Critic | 리뷰, 평가, 판정 | 직접 코드 수정 |
