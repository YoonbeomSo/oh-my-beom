Architecture Overview

4개 진입점 스킬이 에이전트 팀을 강제 실행하여, plan 작성부터 커밋까지 전체 개발 사이클을 자동화합니다. 빌드/테스트 실패 시 자동 교정 루프, 에러 자동 기록, 위험 명령 물리적 차단으로 구조적 안전을 보장합니다.

6 Phase 파이프라인

1 Setup: 서버 빌드/기동 + Playwright 확인 + 계정 조회
2 OTP 바이패스: 패턴 탐지 → 자동 주석 처리 (config 기반)
3 기존 테스트 확인: → MODE 결정 (실행만 / 생성+실행)
4 테스트 실행: web-test-runner (8 tools) 또는 web-tester (16 tools)
5 통과 마커 생성: .dev/web-test-passed (커밋 게이트)
6 정리: 서버 종료 + OTP 원복 + 결과 보고 (무조건 실행)
OTP 안전 장치: 수정 전 git stash → 파일 목록 .dev/otp-bypass-files.txt 기록 → 테스트 완료 후 git checkout 원복 → 원복 검증

Prohibition Rules (CLAUDE.md)

  • PR 머지 금지 — gh pr merge 실행 금지. PR 링크만 제공
  • 보호 브랜치 직접 커밋 금지 — 작업 브랜치 먼저 생성
  • Co-Authored-By 금지
  • 팀 실행 생략 금지 — "간단하다"는 이유로 생략 불가
  • plan 파일 생략 금지 — docs/plan/plan_{작업}.md 필수 생성
  • QA 호출 생략 금지 — 모든 구현 후 반드시 QA 디스패처(Codex 또는 Claude qa-manager)로 리뷰. 오케스트레이터 자체 리뷰 대체 금지
  • 팀 에이전트 복구 스킬 호출 생략 금지 — TeamCreate 직후 환경 감지 후 tmux-team-agent 또는 cmux-team-agent 호출
  • QA 루프 생략 금지 — Critical 발견 시 반드시 수정
  • [WEB-TEST-REQUIRED] 무시 금지 — 마커 감지 시 질문 없이 즉시 웹 테스트 실행
  • 민감 파일 커밋 금지 — .env*, *.key, *.pem, credentials*
  • 환각 금지 — 존재하지 않는 API/패키지/경로 지어내기 금지

Behavior Rules

1. 코딩 전에 생각하라
읽기 우선, 가정 명시, 환각 금지
2. 단순함 우선
최소 코드, 불필요한 추상화 금지
3. 외과적 변경
무관한 라인 수정 금지
4. 점진적 실행
작은 단위, 중간 검증
5. 실패 대응
근본 원인 분석, 반복 금지
6. 피드백 반영
1회 → MEMORY, 2회 → rules 승격. error-log 사전 회피. 3회 반복 → 대안 탐색
7. 프로세스 생략 금지
"변경이 작다" 등의 합리화로 QA/team-agent 생략 불가
