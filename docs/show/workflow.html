1
/lens — 정책가
비즈니스 정책 영향도 분석
언제 사용: 비즈니스 규칙, 정책, 법적 요구사항이 관련된 변경 시

키워드: 정책 비즈니스 lens 영향도

내부 에이전트: architect (복잡도 분석)

산출물: 정책 영향 보고서 (기존 정책 매핑, 영향 범위, 리스크)
결제 한도 변경이 기존 정책에 미치는 영향 분석해
FDS 규정 변경 시 영향도 확인해줘
2
/research — 조사관
코드베이스 탐색 + 기술 조사
언제 사용: 코드 구조를 모르거나, 기술적 맥락 파악이 필요할 때

키워드: 분석해 조사해 research 리서치

3-Phase: 탐색 → 패턴 수집 → 분석 보고

산출물: 조사 보고서 (코드 구조, 의존성, 기존 패턴, 기술적 제약)
현재 인증 모듈 구조 분석해
결제 서비스 의존성 조사해줘
3
/plan — 계획자
4-Mode 계획 수립 (Direct / Interview / Consensus / Review)
언제 사용: 큰 기능, 여러 선택지 비교, 팀과 공유할 계획이 필요할 때

키워드: 계획 plan 설계해 플랜

4가지 모드:
Direct
명확한 요구사항
Interview
애매한 요구사항
Consensus
다관점 합의
Review
기존 계획 검토
산출물: docs/plan/{이슈키}-plan.md (TODO + 완료 기준 + 검증 태그)
[JIRA-789] 인증 시스템 JWT 전환 계획 세워줘
결제 리팩토링 설계해
4
/dev — 개발자
전체 개발 사이클 (PRD → 설계 → 구현 → 리뷰 → PR)
언제 사용: 실제 코드를 작성/수정하고 PR을 만들 때

키워드: 개발해 dev 구현해 만들어줘

6 Phase: Setup → Requirements → Design → Implement → Review → Complete
13 에이전트: 하이브리드 모델 (Task + Agent teammate)
검증 루프: TeammateIdle + TaskCompleted 훅으로 자동 품질 게이트
