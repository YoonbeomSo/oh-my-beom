훅 (6종)

훅은 Claude Code 이벤트에 반응하여 자동 실행되는 셸 스크립트다. 동작을 차단하거나, 컨텍스트를 주입하거나, 정책을 강제할 수 있다.

SessionStartsession-start
진행 중인 계획(IN_PROGRESS)을 감지하고 워크플로우 가이드를 표시한다.

트리거

세션 시작, /clear, /compact

동작

  • docs/plan/에서 IN_PROGRESS 계획 파일 탐색
  • 있으면: 계획 요약 + 재개 안내 표시
  • 없으면: 워크플로우 가이드 테이블 표시
UserPromptSubmitprompt-router
사용자 입력에서 키워드를 감지하여 적절한 스킬로 자동 라우팅한다.

정체 감지 및 에스컬레이션

/dev 오케스트레이터가 4가지 정체 패턴을 감지하고 전문 스킬로 에스컬레이션한다.

패턴감지 기준1차 대응2차 대응

설정 참조 (config.json)

issueKey — 이슈 키 패턴

패턴: ^[A-Z]+-[0-9]+$ (예: JIRA-123, AFS-6)

사용처: /dev (브랜치명 생성), /commit (커밋 메시지 접두사)

projectTypes — 언어별 빌드/린트/테스트 명령
타입감지 파일빌드린트테스트
kotlin-javabuild.gradle(.kts)./gradlew build./gradlew ktlintFormat./gradlew test
nodejspackage.jsonnpm/bun run buildnpm/bun run lint --fixnpm/bun test
pythonpyproject.toml, setup.pypython -m buildruff format + checkpytest --tb=short

사용처: /dev (phase-setup 타입 감지, phase-review Mechanical Gate), architect, coder

verification — 계획 검증 설정
키값효과
enabledtrue계획 파일 생성 + TODO 검증 활성화
autoVisualizetrue각 Phase 완료 후 HTML 대시보드 자동 생성
mechanicalChecks{lint, test, build: true}review Phase에서 린트/테스트/빌드 게이트 활성화
maxVerificationRetries2FAILED TODO 항목 최대 재시도 횟수

사용처: /dev, task-verifier 훅, idle-checker 훅, /todo-verifier, /plan-visualizer

uiTesting — E2E 브라우저 테스트 설정
키값효과
enabledtruereview Phase에서 브라우저 UI 테스트 활성화
frontendFilePatterns*.html, *.jsx, *.tsx, *.vue...프론트엔드 변경 감지 패턴
devServerUrlhttp://localhost:3000기본 개발 서버 URL
maxHealerRetries2test-heal 최대 재시도 횟수

사용처: /dev (phase-review step 2.5), /web-test-qa

contextLimits — 역할별 최대 입력 라인 수
