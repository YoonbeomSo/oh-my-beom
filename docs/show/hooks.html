S
session-start
SessionStart · 활성 계획 감지
트리거: 세션 시작, clear, compact 시
매처: startup|clear|compact
실행 방식: 동기 (async: false)

동작 흐름:
세션 시작 → docs/plan/*-plan.md 탐색 → IN_PROGRESS 계획? → 활성 계획 컨텍스트 주입
감지 정보: 계획 제목, TODO 진행률 (done/total), 잔여 항목 수, 파일 경로

활성 계획 있으면: [ACTIVE PLAN DETECTED] + 진행 상황 요약 → 이전 세션 이어가기 안내
활성 계획 없으면: suppressOutput: true — 아무 개입 없이 통과
R
prompt-router
UserPromptSubmit · 키워드 감지 → 스킬 라우팅
트리거: 모든 사용자 프롬프트 (matcher: *)
타임아웃: 3초

동작 흐름:
사용자 프롬프트 → 소문자 변환 → 키워드 매칭 → 스킬 라우팅 컨텍스트
매칭 규칙 (우선순위 순):
계획|plan|설계해|플랜 → /plan
분석해|조사해|research|리서치 → /research
끝까지|멈추지마|완료할때까지|persist → /persist
개발해|dev |구현해|만들어줘 → /dev
정책|비즈니스|lens|영향도 → /lens
미매칭 시: {"continue": true, "suppressOutput": true} — 아무 개입 없이 통과
G
pre-tool-guard
PreToolUse:Bash · 보호 브랜치 커밋 차단
트리거: Bash 도구 사용 시 (matcher: Bash)
타임아웃: 2초

동작 흐름:
Bash 명령어 → git commit 포함? → 보호 브랜치? → DENY
보호 대상 브랜치:
main master develop

차단 시 응답: permissionDecision: "deny" + 작업 브랜치 생성 안내 메시지
비-git commit 명령: {"continue": true} — 통과
V
task-verifier
TaskCompleted · 계획 완료 기준 검증 (품질 게이트)
트리거: 태스크 완료 시
타임아웃: 300초 (5분)
비활성화: config.json → verification.enabled: false

동작 흐름:
태스크 완료 → docs/plan/*-plan.md 탐색 → IN_PROGRESS 계획 파싱 → 완료 기준 검증 → PASS / BLOCK
검증 항목:
  • TODO 미완료 TODO 항목 (- [ ] [TODO-*])
  • LINT 린트 통과 여부 (프로젝트 타입별 명령어)
  • TEST 테스트 통과 여부
  • BUILD 빌드 통과 여부
  • FILE_EXISTS 필수 파일 존재 여부
  • CRITERIA 기계적 검증 불가 — todo-verifier 에이전트 위임
차단 시: exit 2 + stderr로 미충족 항목 피드백 → 작업 계속 진행 지시
통과 시: exit 0 — 태스크 완료 허용
I
idle-checker
TeammateIdle · 미완료 TODO 자동 할당
트리거: 팀메이트 유휴 상태 감지 시
타임아웃: 10초
비활성화: config.json → verification.enabled: false

동작 흐름:
팀메이트 유휴 → IN_PROGRESS 계획 탐색 → 미완료 TODO 수집 → 다음 작업 지시
수집 정보: TODO ID, 설명, 소속 Step
표시 제한: 최대 5개 (초과 시 "외 N개" 표시)
미완료 항목 없으면: IDLE_OK — 유휴 허용
미완료 항목 있으면: exit 2 + stderr로 다음 작업 목록 전달
